博客搭建记录
更新于 2020-02-27 总结 记录

 起因

我在csdn写了博客1年多了，翻找自己以前的文章发现几个问题

  • 内容杂乱不成体系
  • 深度不够
  • 长篇幅文章不合适

简单来说，自己想提升一点，用新的博客重新开始记录，csdn当然还会用，只是考虑到自己要记录一些内网相关的学习，以及对漏洞的具体分析，csdn的文章形式和主题我都觉得不是很合适。

对于长篇幅文章，比如我在csdn中记录Linux学习，和web开发基础的文章，是以目录单篇置顶，各个章节从目录直接进入，解决了文章发布多很难找到自己文章的问题，所以对于长篇文章我可能还是会放到csdn上。

 github+hexo+window

选择github是因为我有个比较喜欢的Annie主题在github上能用hexo来搭建。

我有一个阿里云学生机但考虑到迁移、域名，还有自己经常恢复快照，这是考虑到安全性了，就选择了在github上去搭建。

wordpress是我考虑过的，wordpress有专门插件迁移方便，数据大了，要付费，个人博客内存不大估计不用，而且自己迁移也不难。wordpress也不用 使用图床来存储图片。

但考虑到wordpress放不到GitHub上是用不了Annie主题，页面内嵌广告也不少，还是放弃了


## 一台电脑部署两个或多个GitHub 账号

以我的举例

  • 账户一：IlHlI
  • 账户二：zalcat（博客现在使用的账户）

 密钥配置

ssh-keygen -t rsa -C "这里是你的邮箱"

配置新密钥名称默认是id_rsa,
两个账户不能相同，比如新建的为，id_rsa_github

我的配置

 config配置

查看你的.ssh/根路径下, 有没有config文件,（ 比如我的路径为C:\Users.ssh）没有则命令创建一个config文件

ps:没有后缀全名的config文件

写入如下

#第一个账号，默认使用的账号，不用做任何更改
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa

#第二个新账号，#"xxxxxx"为前缀名，可以任意设置，要记住，后面需要用到
Host xxxxxx.github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/这里是你创建的新密钥的名称

 取消全局用户名/邮箱，单独设置

执行如下命令，取消全局用户名和邮箱配置（如果已经设置了全局的话）：

git config --global --unset user.name
git config --global --unset user.email

分别进入你的两个Hexo博客.git目录下执行以下命令单独设置用户名/邮箱：

git config user.name "用户名"
git config user.email "邮箱"

 hexo 配置文件修改git地址

打开你的第二个博客Hexo目录下的_config.yml文件，找到deploy关键字，写入以下配置并保存：

deploy:
  type: git
  repository: git@xxxxxx.github.com:你的用户名/你的用户名.github.io.git
  branch: master

比如我的配置：

deploy:
  type: git
  repo: git@zalcat.github.com:zalcat/zalcat.github.io.git
  branch: master

再次执行hexo g -d就能成功将新的博客部署到 Github 上了。

 修改主题

GitHub clone下来的主题Annie 主打的是文艺、优雅、简洁。

我首先对其字体，背景做了调整，发现如下问题

Annie.css是我要修改css用到的唯一文件。

执行hexo clean 的时候会清除.deploy_git下的Annie.css，hexo g又生成 .deploy_git下的文件，但不论我是先修改哪个还是一起修改，执行hexo g -d推送到github上时，都会恢复为主题原来的配置，也就是我的修改没有生效又回去了。

最后的办法是，对github上和本地这两个Annie.css同时做修改，hexo g -d 成功修改

 图床的使用

 在本地写md文档时候插入图片是一件挺麻烦的事情，使用本地图片的话，每次在给别人分享文档或者上传博客的时候就比较难顶了。网上有两种方法可以解决这个问题，一种是使用图片的base64编码，但是图片转换为base64编码之后，编码长度会很大，文档的可读性不好，而且也会使文档的体积急剧变大，这种方法显然不太好。另外一种常用的方法就是使用图床

下面就说一下搭建图床的一个坑。

首先阿里云购买和配置图床Bucket，而后使用PicGo工具来加快图片的使用

PicGo下载地址https://github.com/Molunerfinn/PicGo

这类工具要有接入阿里云的权限，才完成把图片上传到Bucket然后把图片的链接返回，但是账户的直接权限太大，考虑到安全起见，在账户中再新建一个账户给他的权限仅限于OSS ,而没有ECS、控制台等。

但PicGo我是用小权限用户的时候出现无法连接，错误报告日志，记录的是Accesskeyid无此记录

我下载了阿里云官方的oss链接client，却可以上传成功。考虑到picGo方便一点，就直接再账户下创建了Accesskey,连接成功。

 The end

以上就是博客搭建遇到的问题，后续肯定会出现一些现在没考虑到，没出现的弊端，但办法总比困难多。

希望从继csdn后开辟的新博客天地，zalcat博客能成为更高质量、更有深度、更具个性的博客。
